RESTful 风格

RESTful 是一种风格，在 RESTful 中，一切都被认为是资源，每个资源有对应的 URL 标识.

不是标准也不是协议，只是一种风格。当然你也可以不按照他的风格去写

1. 接口 url：​

<!-- 传统接口 -->

http://localhost:8080/api/get_list?id=1

http://localhost:8080/api/delete_list?id=1

http://localhost:8080/api/update_list?id=1

<!-- RESTful 接口 -->

http://localhost:8080/api/get_list/1 查询 删除 更新

RESTful 风格一个接口就会完成 增删改差 他是通过不同的请求方式来区分的

1. 查询 GET
2. 提交 POST
3. 更新 PUT PATCH
4. 删除 DELETE

2. nestjs 中 RESTful 版本控制：​

1. main.ts：​

import { NestFactory } from "@nestjs/core";
import { VersioningType } from "@nestjs/common"; // 导入 预设值
import { AppModule } from "./app.module";

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  // 配置版本类型
  app.enableVersioning({
    type: VersioningType.URI,
  });
  await app.listen(3000);
}
bootstrap();
值描述
URI版本将在请求的 URI 中传递（推荐）
Header自定义请求标头将指定版本
Media Type请求的 Accept 标头将指定版本

2. controller.ts：​

Controller 变成一个对象 通过 version 配置版本（接口前面加版本号）：

import { Controller, Get, Post, Body, Patch, Param, Delete, Version } from '@nestjs/common';
import { UserService } from './user.service';
import { CreateUserDto } from './dto/create-user.dto';
import { UpdateUserDto } from './dto/update-user.dto';

// 请求地址变为：/v1/user
@Controller({
  path:"user",
  version:'1'
})
export class UserController {
  constructor(private readonly userService: UserService) {}

  @Post()
  create(@Body() createUserDto: CreateUserDto) {
    return this.userService.create(createUserDto);
  }

  @Get()
  findAll() {
    return this.userService.findAll();
  }

  @Get(':id')
    // @Version('1') 单接口 配置版本 请求地址变为：/v1/user/:id
  findOne(@Param('id') id: string) {
    return this.userService.findOne(+id);
  }

  @Patch(':id')
  update(@Param('id') id: string, @Body() updateUserDto: UpdateUserDto) {
    return this.userService.update(+id, updateUserDto);
  }
