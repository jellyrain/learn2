类型约束

1. 类型约束 使用方式：​

在变量后面添加 : 和类型 限制这个变量可以赋值的类型

const str: <type> = xxx
const str: string = "hello world";

2. 基础类型：​

JS 基础类型：Boolean、Number、String、null、undefined 以及 ES6 的 Symbol 和 ES10 的 BigInt

1. 数字类型允许值：​

NaN、普通数字、Infinity（无穷大）、十进制、十六进制、八进制、二进制

2. void 和 undefined 和 null 最大的区别：​

  1. 与 void 的区别是，undefined 和 null 是所有类型的子类型，也就是说 undefined 类型的变量，可以赋值给 string 类型的变量

  2. 开启严格模式， null 不能 赋予 void 类型：

    配置了 tsconfig.json 开启了严格模式：

    {
      "compilerOptions": {
        "strict": true
      }
    }

3. any 类型 和 unknown 顶级类型：​

  1. any：失去了 TS 类型检测的作用
  2. unknown 类型也被认为是 top type ，但它更安全。与 any 一样，所有类型都可以分配给 unknown

区别：

  1. any：和写 JS 是没有任何区别的
  2. unknown：如果是对象，不允许调用内部 属性 和 方法
  3. unknown 可赋值对象只有 unknown 和 any

3. 接口 与 对象类型：​

1. 接口：​

象的方式要用关键字 interface（接口），我的理解是使用 interface 来定义一种约束，让数据的结构满足约束的格式

interface Person {
    b:string,
    a:string

const p: Person =  {
    a = 'a',
    b = 'b'
}

注意：

  1. 使用接口约束的时候 不能多一个属性 也不能少一个属性（默认情况）
  2. 如果有重名的 interface接口 使用时候 必须同时满足全部约束
  3. ? 可选属性：
    1. 选属性的含义是该属性可以不存在
    2. 示例：b?:string
  4. [propName: string] 任意属性：
    1. 一旦定义了任意属性，那么确定属性和可选属性的类型都 必须是它的类型的子集
    2. 示例：[propName: string]: any
  5. readonly 只读属性：
    1. 只读属性是 不允许被赋值 的只能读取
    2. 示例：readonly a: string

4. 数组类型：​

1. 基础写法：​

基础类型 + [] 表示 数组类型

const arr: number[] = [1, 2, 3]; //数字类型的数组
const arr2: string[] = ["1", "2"]; //字符串类型的数组
const arr3: any[] = [1, "2", true]; //任意类型的数组

2. 泛型写法：​

const arr: Array<number> = [1, 2, 3, 4, 5];

3. 多维数组：​

const arr: number[][] = [[], []];

const arr: Array<Array<number>> = [[], []];

4. 用接口表示数组：​

一般用来描述类数组

interface NumberArray {
  [index: number]: number;
}
const fibonacci: NumberArray = [1, 1, 2, 3, 5];
//表示：索引的类型是数字时，那么值的类型必须是数字

5. 函数：​

1. 普通参数约束：​

//注意，参数不能多传，也不能少传 必须按照约定的类型来
const fn = (name: string, age: number): string => {
  return name + age;
};

2. 接口参数约束：​

//定义参数 num 和 num2  ：后面定义返回值的类型
interface Add {
  (num: number, num2: number): number;
}

const fn: Add = (num: number, num2: number): number => {
  return num + num2;
};

interface User {
  name: string;
  age: number;
}

function getUserInfo(user: User): User {
  return user;
}

3. 可选参数 和 默认值：​

//通过?表示该参数为可选参数
const fn = (name: string, age?: number): string => {
  return name + age;
};

const fn = (name: string = "我是默认值"): string => {
  return name;
};

6. 特殊类型：​

1. 联合类型：​

使用 | 同时满足多种类型

let myPhone: number | string = "010-820";

2. 交叉类型：​

使用 & 多种类型的集合，联合对象将具有所联合类型的所有成员

interface People {
  age: number,
  height： number
}

interface Man{
  sex: string
}

const xiaoman = (man: People & Man) => {
  console.log(man.age)
  console.log(man.height)
  console.log(man.sex)
}

3. 类型断言：​

语法：值 as 类型 或 <类型>值 value as string <string>value

类型断言只能够「欺骗」TypeScript 编译器，无法避免运行时的错误

1. any 临时断言：​

window.abc = 123;
//这样写会报错因为window没有abc这个东西

(window as any).abc = 123;
//可以使用 any 临时断言在 any 类型的变量上，访问任何属性都是允许的

2. as const 断言：​

let names2 = "小满" as const;
names2 = "aa"; //无法修改

3. 类型断言是不具影响力的：​

并没有什么用 并不会影响结果, 因为编译过程中会删除类型断言
